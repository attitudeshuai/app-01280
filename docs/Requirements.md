**版本**: 1.0  
**创建日期**: 2026-02-02  
**状态**: 待确认

---

## 📌 1. 业务目标

构建一个现代化的**电影票在线预订平台**,支持用户便捷选座购票,并提供**创新陪伴功能**(玩偶/路人匹配),提升单人观影体验。

---

## 🎯 2. 核心功能需求

### 2.1 用户认证系统
**鉴权方案**: SaToken  
**功能点**:
- 用户注册(手机号/邮箱)
- 用户登录(Session + Token)
- 权限控制(普通用户/管理员)
- 登录状态保持

### 2.2 标准购票流程 (必需)

```
打开平台
  ↓
登录/注册
  ↓
选择影院 (支持城市/地区筛选)
  ↓
筛选影片与场次 (按时间/类型/热度)
  ↓
查看座位图并选座 (可视化座位选择)
  ↓
确认订单信息 (价格/座位/场次)
  ↓
支付 (支付宝/微信模拟)
  ↓
生成电子取票码 (二维码)
  ↓
到店取票 (核销码)
```

**核心交互点**:
- 实时座位状态同步(已售/可选/锁定)
- 订单超时取消机制(15分钟未支付)
- 支付失败重试逻辑

### 2.3 特色陪伴功能 (创新点)

**触发条件**: 选座后,若座位为单数,系统提示"是否需要陪伴?"

**陪伴类型**:
1. **玩偶陪伴**:
   - 用户从预设玩偶列表中选择(小黄鸭/熊猫/龙猫等)
   - 系统预留邻座并标记为"玩偶占座"
   - 影院提供实体玩偶放置

2. **随机路人匹配**:
   - 系统匹配同场次同时段的单人观众
   - 双方确认后共享邻座信息
   - 保护隐私(仅显示昵称/头像)

**数据逻辑**:
- 匹配池管理(性别/年龄段可选筛选)
- 匹配超时策略(5分钟无响应则降级为普通座位)

## 🏗️ 3. 技术栈要求

### 后端
- **框架**: Spring Boot 3.x
- **鉴权**: SaToken (Session + Redis)
- **数据库**: MySQL 8.0+
- **缓存**: Redis (座位锁/Token存储)
- **API风格**: RESTful

### 前端
- **框架**: Vue 3 + Vite
- **UI库**: Element Plus / Ant Design Vue (满足"Dribbble标准")
- **状态管理**: Pinia
- **路由**: Vue Router

### 基础设施
- **容器化**: Docker + Docker Compose
- **反向代理**: Nginx (前端静态资源)
- **数据库迁移**: Flyway / Liquibase

---

## 📊 4. 数据模型概要

### 核心实体
1. **User** (用户)
   - 基础信息 (手机/邮箱/昵称/头像)
   - 会员等级

2. **Cinema** (影院)
   - 名称/地址/城市
   - 影厅列表

3. **Movie** (影片)
   - 片名/类型/时长/海报
   - 评分/简介

4. **Showtime** (场次)
   - 影片 + 影厅 + 时间
   - 价格

5. **Seat** (座位)
   - 座位号(行-列)
   - 状态(可选/已售/锁定/玩偶占座)

6. **Order** (订单)
   - 用户 + 场次 + 座位列表
   - 支付状态/取票码

7. **Companion** (陪伴记录)
   - 类型(玩偶/路人)
   - 匹配状态/玩偶类型/匹配用户ID

## 🚢 6. 交付标准

### Docker Compose 验证要求
```bash
# 用户操作流程
docker compose up -d        # 启动所有服务
# 访问 http://localhost:8082  → 用户端前端
# 访问 http://localhost:8081  → 管理后台前端
# 访问 http://localhost:8080  → 后端API
```

**服务清单**:
- `frontend`: 用户端Vue应用(Nginx, 端口8082)
- `admin`: 管理后台Vue应用(Nginx, 端口8081)
- `backend`: Spring Boot API (端口8080)
- `mysql`: 数据库(含初始化脚本)
- `redis`: 缓存服务

### 功能完整性验证
- [ ] 用户可注册并登录
